# Продажи и склады — MVP

Веб-приложение для учёта продаж и складов по городам Казахстана.

## Стек

- Next.js (App Router) + TypeScript + TailwindCSS
- Supabase (Auth + Postgres + RLS)

## Быстрый старт

### 1. Создать проект Supabase

1. Перейдите на [supabase.com](https://supabase.com) и создайте проект
2. В настройках проекта скопируйте URL и anon key

### 2. Настроить переменные окружения

Создайте `.env.local` в корне проекта:

```
NEXT_PUBLIC_SUPABASE_URL=https://ваш-проект.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=ваш-anon-key
```

### 3. Выполнить SQL миграцию

1. Откройте Supabase Dashboard → SQL Editor
2. Скопируйте содержимое файла `supabase.sql`
3. Выполните запрос

### 4. Включить Email Auth

1. Supabase Dashboard → Authentication → Providers
2. Убедитесь, что Email включён

### 5. Создать admin-пользователя

1. Authentication → Users → Add user → Create new user
2. Введите email и пароль (например: admin@test.com)
3. Скопируйте **User ID** (UUID)

4. SQL Editor → выполните:
```sql
INSERT INTO profiles (id, display_name, role, is_active)
VALUES ('вставьте-user-id-сюда', 'Admin', 'admin', true);
```

### 6. Установить зависимости и запустить

```bash
npm install
npm run dev
```

Откройте http://localhost:3000 и войдите с email и паролем admin.

## Инструкция по тестированию MVP

### Подготовка данных

1. **Создать город**
   - Войти как admin → Склады
   - Нажать «+ Город», ввести название (например: Алматы), сохранить

2. **Создать товар**
   - Товары → «+ Создать»
   - Название: Тестовый товар, Цена: 5000, Сохранить

3. **Добавить остаток на склад**
   - Склады → «Приход товара»
   - Выбрать город (Алматы), товар, количество (например 10)
   - Добавить

4. **Создать продавца**
   - Authentication → Users → Add user
   - Email: seller@test.com, пароль: любой
   - Скопировать User ID
   - Продавцы → «Привязать пользователя»
   - User ID, Display name: Продавец 1, Роль: Продавец, Город: Алматы
   - Сохранить

### Тест сценария продавца

1. Выйти, войти как seller@test.com
2. **Новый заказ**
   - Нажать «Новый заказ»
   - Добавить «Тестовый товар» в корзину
   - Количество 2, выбрать тип оплаты (Наличные)
   - Подтвердить

3. **Проверить списание**
   - Остатки → должно остаться 8
   - Сегодня: сумма 10000, заказов 1

4. **Отменить заказ**
   - Мои заказы → открыть заказ → Отменить (в течение 10 минут)
   - Остатки снова 10

### Тест админки

1. Войти как admin
2. Дашборд: выручка по городам, низкий остаток
3. Заказы: фильтры, просмотр, отмена
4. Товары: создание, редактирование, архивация (is_active=false)
5. Склады: города, приход, корректировка
6. Продавцы: привязка пользователя, редактирование профиля
